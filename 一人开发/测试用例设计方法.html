一、等价类划分法

核心思想

将输入数据划分为若干等价类，每个等价类中的数据对于揭露程序错误具有等效作用。从每个等价类中选取代表性数据进行测试。

有效等价类

满足需求规格说明的合理输入。如：用户名 6-20 位字母数字。

无效等价类

不满足需求规格说明的不合理输入。如：空值、特殊字符、超长。

等价类划分法 - 用户注册案例

需求：用户名 6-20 位字母或数字，密码 8-16 位。

类型测试数据预期结果
有效6位字母数字（user01）注册成功
有效20位字母数字注册成功
无效少于6位（abc）提示用户名长度不足
无效超过20位提示用户名过长
无效特殊字符（user@#$）提示格式错误
无效空值提示不能为空

二、边界值分析法

核心思想

在等价类划分基础上，针对边界条件进行测试的方法。错误往往发生在输入范围的边界上。

边界值选取

  • 下边界：a-1, a, a+1
  • 上边界：b-1, b, b+1

案例：年龄输入（1-120岁）

无效下边界下边界+1…上边界-1上边界无效

三、错误推测法

定义

基于经验和直觉，推测程序可能存在的缺陷，有针对性地设计测试用例。

核心思想

站在破坏者角度思考程序漏洞。

常见错误推测点

错误场景检查要点
空值/NULL处理检查空指针异常
并发操作检查线程安全
重复提交检查防重机制
超时处理检查资源释放
网络异常检查超时重试
数据格式异常检查格式校验

四、判定表法

定义

针对多条件组合逻辑，通过判定表穷举所有条件组合，确保测试覆盖完整。

适用场景

多条件组合、逻辑复杂的业务规则。

案例：电商优惠计算

规则：VIP会员享8折 · 订单满200减20 · VIP且满200 8折后再减20。

条件规则1规则2规则3规则4
是否VIPYYNN
订单≥200YNYN
实付金额8折-208折原价-20原价

五、因果图法

定义

分析输入条件（因）和输出结果（果）之间的关系，用图形化方式表达逻辑关系。

适用场景

输入输出关系复杂的逻辑判断。

基本符号

符号名称含义
恒等 =恒等原因为真，结果为真
非 ~非原因为真，结果为假
与 ^与所有原因都为真，结果为真
或 v或任一原因为真，结果为真

六、正交实验法

定义

利用正交表选取代表性组合，用最少用例实现较高覆盖。

适用场景

多因素多水平，全组合数量过大无法穷举。

核心公式

Ln(m^k)：n = 实验次数，m = 水平数，k = 因子数。

案例：手机兼容性测试

因素水平
系统（4个）Android 11/12、iOS 15/16
屏幕（3个）小屏 / 中屏 / 大屏
网络（3个）WiFi / 4G / 5G
  • 全组合：36 种
  • 正交表测试：9 种

七、场景法

定义

从用户角度模拟实际操作场景，覆盖基本流和备选流。

  • 基本流：正常的操作流程
  • 备选流：异常或分支流程

案例：ATM取款流程

  • 基本流：插卡 → 输入密码 → 选择取款 → 输入金额 → 取款成功
  • 备选流：卡片无效、密码错误3次、余额不足、超过限额、用户取消

八、状态迁移法

定义

分析状态之间的转换条件，设计测试覆盖所有状态转换。覆盖所有状态转换路径，确保每个转换条件都被测试。

适用场景

订单状态、工作流、协议状态机。

案例：订单状态流转

  • 正向：待支付 → 待发货 → 待收货 → 已完成
  • 异常分支：已取消、已退款

方法选择指南

方法适用场景特点优先级
等价类划分输入域明确减少测试用例数量高
边界值分析有明确边界发现边界错误高
错误推测经验丰富针对性强中
判定表法多条件组合逻辑覆盖完整中
因果图法因果关系复杂可视化逻辑低
正交实验法多因素多水平高效覆盖高
场景法业务流程用户视角高
状态迁移法状态转换状态覆盖中

组合使用建议

  1. 等价类划分 + 边界值分析：处理输入域，减少用例数量，发现边界错误。
  2. 判定表 / 因果图：处理逻辑组合，确保条件覆盖完整。
  3. 场景法：覆盖业务流程，从用户视角设计测试。
  4. 状态迁移法：覆盖状态流转，确保状态转换路径完整。
  5. 错误推测法：补充异常场景，基于经验发现潜在缺陷。
组合使用效果更佳。

总结

  1. 没有万能的方法：根据具体场景选择最合适的测试方法。
  2. 组合使用效果更佳：多种方法结合，实现更全面的测试覆盖。
  3. 持续补充测试用例库：积累经验，不断完善测试用例库。
  4. 关注业务价值：优先测试核心功能，保障业务价值。

综合实践题

需求场景：电商购物车结算

练习要求

  1. 等价类 + 边界值：针对商品数量输入设计测试用例。
  • 商品数量：1-99 件，支持加减按钮
  • 支付方式：微信 / 支付宝 / 银行卡 / 货到付款
  • 优惠规则：VIP 享 8 折，满 200 免运费
  • 库存限制：商品A库存 50 件
  1. 判定表法：支付 + 会员 + 金额的组合测试。
  2. 场景法：购物车结算基本流和备选流。
  3. 错误推测法：补充可能的异常测试场景。

四、错误推测法 - 错误分类（25分）

输入相关错误（2分）

  • 超长字符输入
  • 负数输入
  • 科学计数法
  • Unicode数字
  • 前导零

业务逻辑错误（3分）

  • 多张优惠券
  • 优惠大于商品
  • 运费计算错误
  • 折扣叠加错误
  • 商品删除残留

并发/性能错误（2分）

  • 多设备修改
  • 库存超卖
  • 快速连续点击
  • 支付时库存变化

网络/环境错误（2分）

  • 弱网环境
  • 网络中断重连
  • Session过期
  • 支付回调丢失

安全相关错误（1分）

  • 价格篡改
  • 优惠券重复使用
  • 越权访问
  • SQL注入
  • XSS攻击

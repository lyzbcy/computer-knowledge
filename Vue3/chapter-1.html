第一章 · Vue3 入门 —— 选项式 API 起手式

声明式渲染 + 响应式 + 模板语法 ｜ 本项目强制选项式 API｜含 … 道互动关卡

01 · 学习目标

学完这一章你要能做到

  • 用一句话说清 Vue 是什么，并指出它的两个核心功能（声明式渲染、响应式）。
  • 默写一个选项式 API 组件的三件套：data()、methods、mounted，并说出它们各自的职责。
  • 区分五种模板写法：文本插值、属性绑定、事件绑定、双向绑定、原始 HTML，并知道对应的指令。
  • 解释响应式：改了 this.count，为什么页面上的数字会自动变。
  • 能指出至少 2 个入门误解（如「把组合式和选项式混用」「以为 {{ }} 能写任意 JS」）。
阅读路径：背景包 → Vue3 结构地图 → 核心判断 → 关键概念 → 做关卡题 → 生成报告。
项目约束

老田的硬性要求：本项目采用「选项式 API」，禁止使用「组合式 API」。学完后你应能解释为什么团队要统一风格，并识别一份代码是哪种 API。这一章全程围绕选项式展开。

原文依据：《初心与使命》——「我们采用的是"选项式"API，不是组合式，为保持风格统一，禁止使用"组合式"API」。
02 · 零基础背景包

读这章前先知道的 4 件事

背景①什么是「框架」

写网页本质是操作 HTML（结构）+ CSS（样式）+ JavaScript（行为）。原生 JS 要你手动找到元素、手动改内容（document.getElementById 之类）。框架帮你把这些重复劳动抽象掉，给你一套规则，照着写就行。Vue 就是干这个的。

读这章需要它：否则你不知道「为什么有了 JS 还要 Vue」。
背景②什么是 DOM

DOM（Document Object Model）= 浏览器把 HTML 解析成的一棵「元素树」。页面上看到的每个标签，都是这棵树上的一个节点。所谓「更新页面」，本质就是改这棵树上的节点。Vue 帮你改 DOM，你只管改「数据」。

读这章需要它：响应式就是「数据变 → Vue 自动改 DOM」。
背景③什么是「响应式」（先有个直觉）

普通变量改了，页面不会变；响应式变量改了，页面自动跟着变。你可以把它想成「数据被装了监控」——谁改它，监控就通知页面刷新。具体原理本章后面讲。

这是 Vue 区别于「写死页面」的根本能力，不懂它就等于没懂 Vue。
背景④选项式 vs 组合式 —— 两种「写组件的风格」

同一个 Vue3 组件，可以写成两种长相：

  • 选项式 API（Options API）：把代码分门别类塞进 data、methods、mounted 等「选项」里，像一个填表。
  • 组合式 API（Composition API）：用 setup() / <script setup> + ref、reactive 函数式地组织。

两者功能等价，最终产物一样，区别只在「怎么组织代码」。本项目统一用选项式。

不要误解成：组合式比选项式「高级」或「新版」。官方原文：两者只是「不同的 API 风格」，没有高下。
03 · 这章在回答什么

表面 vs 深层问题

表层：Vue3 入门怎么写。

深层：Vue 到底解决了什么问题？它「自动更新页面」的承诺，建立在哪两个核心功能上？作为本项目开发者，我为什么必须走选项式这条路？

只记一个问题：我写了状态和方法，Vue 凭什么能让页面自动跟着变？

带着这个问题读后面所有「核心判断」，你会发现它们都在回答它。
04 · Vue3 结构地图

一张图看懂 Vue3 入门全貌

Vue 官方自我介绍是一棵递进式的能力树——它不是一上来就把全部能力塞给你，而是「需要什么加什么」。读懂这条递进线，就懂了 Vue 的整体形状。

1渐进式框架｜核心承诺：可轻可重，按需引入。
2声明式渲染｜用模板语法「声明」数据和 HTML 的关系，不再手动操作 DOM。
3响应式 ⭐灵魂｜数据一变，Vue 自动更新对应的 DOM。
4模板语法｜{{ }} 文本、v-bind/: 属性、v-on/@ 事件、v-model 双向。
5组件｜把界面拆成可复用的小块，互相组合。
6上手｜脚手架 / SFC（单文件组件 .vue）→ 进真实项目。
节点关系：声明式渲染 + 响应式是两个并列的「核心功能」（官方原话），其余都是从它们长出来的能力。
05 · 核心判断

6 条核心判断（每条都带原文依据）

事实① Vue 是一套「用于构建用户界面」的「渐进式」框架。

这是官方对 Vue 的第一句定性。「构建用户界面」= 做前端 UI；「渐进式」= 不强迫你一次用全部功能。

原文依据：Vue 官网《简介》——「Vue 是一套用于构建用户界面的渐进式框架。」
事实② Vue 的核心功能是两个：声明式渲染 + 响应式。

不是三个、不是一个，是两个并列的核心。后面所有的模板、指令、组件，都建立在这两个之上。

原文依据：Vue 官网《简介》——「两个核心功能：声明式渲染（Declarative Rendering）和响应性（Reactivity）。」
事实③ 声明式渲染 = 你声明「数据和 HTML 的关系」，Vue 替你把 DOM 改好。

对比命令式（你写步骤：先找元素、再改它），声明式只描述「最终要长什么样」，过程交给 Vue。

原文依据：Vue 官网《简介》——「Vue 基于标准 HTML 拓展了一套模板语法，使得我们可以声明式地描述最终输出的 HTML 和 JavaScript 状态之间的关系。」
事实④ 响应式 = Vue 自动跟踪 JS 状态变化，并在变化时响应式地更新 DOM。

这就是「数据变 → 页面自动变」的官方说法。你不用手动调 DOM，改数据即可。

原文依据：Vue 官网《简介》——「Vue 会自动跟踪 JavaScript 状态变化并在改变发生时响应式地更新 DOM。」
事实⑤ 选项式 API 用一个「包含多个选项的对象」描述组件逻辑。

组件本身是一个 JS 对象，它有一组固定「槽位」（选项）：放状态的 data、放方法的 methods、放生命周期的 mounted 等。这些选项里的属性通过 this 暴露在组件实例上。

原文依据：Vue 官网《简介》——「使用选项式 API，我们可以用包含多个选项的对象来描述组件的逻辑，例如 data、methods 和 mounted。选项所定义的属性都会暴露在函数内的 this 上，this 会指向当前组件实例。」
判断⑥ data() 返回的对象会被 Vue 处理成响应式。

这是选项式下「响应式」的入口。你在 data() 里 return 的属性，才具备「改了自动更新页面」的能力；methods 里用 this.xxx 改它，页面就刷新。

原文依据：Vue 官网《响应式基础》（选项式）——组件 data 选项返回的对象，Vue 在创建实例时通过响应式系统将其转换为响应式的；在 methods / 钩子中通过 this 访问和修改这些属性。
06 · 关键概念拆解

6 个关键概念

① 选项式 API（Options API）

定义：用「分门别类的选项对象」组织一个组件。

长这样：

选项式三件套
export default {
  data() {            // ① 状态
    return { count: 0 }
  },
  methods: {          // ② 方法
    increment() {
      this.count++    // 通过 this 访问状态
    }
  },
  mounted() {         // ③ 生命周期：挂载后执行
    console.log('count =', this.count)
  }
}
不要误解成：data 是直接的对象。它必须是一个函数 data()，return 一个新对象——这样每个组件实例才有独立的状态。

② 文本插值 {{ }}

定义：双大括号（Mustache 语法），把数据渲染成纯文本。

<span>{{ message }}</span> 会显示 message 的值。

原文依据：《模板语法》——「文本插值……使用的是"Mustache"语法（双大括号）。」
不要误解成：{{ }} 里能写任意 JS。它是表达式，不是代码块；不能写语句（如 if），且输出永远是纯文本，不是 HTML。

③ 指令（Directives）v-*

定义：带 v- 前缀的特殊 attribute，给元素附加特殊行为。

本章要记住的指令家族：v-bind（属性绑定）、v-on（事件）、v-model（双向）、v-html（原始HTML）、v-if/v-for（条件/循环，后续章）。

原文依据：《模板语法》——「指令（Directives）是带有 v- 前缀的特殊 attribute。」

④ v-bind 与 v-on 的缩写

定义：两个最高频指令都有缩写，正式项目里几乎都用缩写。

全写缩写作用
v-bind:href="url":href="url"属性绑定
v-on:click="fn"@click="fn"事件绑定
原文依据：《模板语法》——「v-bind 缩写」「v-on 缩写」两节。

⑤ v-model（双向绑定）

定义：在表单元素上创建双向数据绑定——输入框打字会更新数据，数据变了也会反映到输入框。

<input v-model="message">：用户输入 → message 变；message 变 → 输入框内容变。

类比：v-bind 是「数据 → 视图」单向水管；v-model 是「数据 ⇄ 视图」双向水管。它帮助理解：为什么表单爱用 v-model。它不能代表：v-model 不是万能绑定，主要用于表单类元素。

⑥ this 在选项式里指向组件实例

定义：在 methods 和生命周期里，this 指向当前组件实例，因此 this.count 能读到 data() 里返回的 count。

易错：箭头函数没有自己的 this，在 methods 里不要用箭头函数定义方法，否则 this 不会指向实例，会读到 undefined。

原文依据：《简介》——选项「所定义的属性都会暴露在函数内的 this 上，this 会指向当前组件实例。」

09 · 误解与边界

4 个最容易踩的坑

误解①：组合式比选项式「高级 / 新」，该优先学

真相：官方原文说它们是「两种不同的 API 风格」，没有高下。本项目明确用选项式，你就照选项式学，别被网上「Vue3 = 组合式」的说法带偏。

误解②：data 是一个普通对象

真相：它必须是函数 data()，返回一个新对象。这样每个组件实例才有各自独立的状态副本；写成普通对象会导致多实例共享同一份状态。

误解③：{{ }} 能输出 HTML

真相：双大括号只渲染纯文本，会原样显示标签符号。要输出 HTML 得用 v-html，且 v-html 有 XSS 风险，不能用来渲染用户输入。

原文依据：《模板语法》——「双大括号将会将数据插值为纯文本，而不是 HTML。」

误解④：methods 里用箭头函数更简洁

真相：箭头函数没有自己的 this，在 methods 中会让 this 指向外层而非组件实例，导致 this.count 报错。methods 必须用普通函数。

边界：本章只覆盖「简介 + 模板语法 + 响应式基础」三页官方文档的范围。v-if/v-for、computed/watch、组件 props、生命周期全集都不在本章——它们在后续章节。别用本章内容强行解释它们。
10 · 背景知识缺口

要完全吃透还差什么

现在就该补（阻碍你写代码）：

  • ES6 模块（export default / import）—— 选项式组件就是用 export default {} 导出的对象。不懂模块语法，第一行就卡住。
  • 箭头函数 vs 普通函数的 this 差异—— 本章 Q8 已埋点，建议手写验证一遍。

可以晚点补（不影响入门）：

  • 虚拟 DOM / 响应式底层（Proxy）—— 知道「自动更新」是怎么实现的。先用起来，再深究原理。
  • 构建工具 Vite / SFC 单文件组件—— 真实项目要 .vue 文件，需配套脚手架。这是「上手」环节，第二章再碰。
下一步追问：「状态变了 Vue 怎么知道哪些 DOM 要更新？」—— 这是响应式原理，留到后续章节。本章只需建立「数据 → 自动更新页面」的因果直觉。

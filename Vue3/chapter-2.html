第二章 · 单文件组件与脚手架

.vue 三段式 ｜ Vite 脚手架 ｜ 本项目SFC 里仍写选项式｜含 … 道互动关卡

01 · 学习目标

学完这一章你要能做到

  • 默写一个 .vue 文件的三段式骨架：<template>、<script>、<style>，并说出每段放什么。
  • 用 npm create vue@latest 起一个项目，跑通 npm run dev，知道开发服务器和热更新是干嘛的。
  • 解释 SFC 与选项式 API 的关系：SFC 是「文件怎么组织」，选项式是「script 里怎么写逻辑」，二者正交。
  • 识别官网示例里的 <script setup>，并知道为什么本项目不写它。
  • 能指出至少 2 个常见误解（如「以为 .vue 必须用 setup」「以为 scoped 是全局隔离」）。
阅读路径：背景包 → 结构地图 → 核心判断 → 关键概念 → 做关卡题 → 生成报告。
项目约束延续

第一章的红线在本章依然有效：本项目采用「选项式 API」，禁止使用「组合式 API」。这意味着，即使你打开 Vue3 官方 SFC 文档看到的示例是 <script setup>，本项目的 .vue 文件里的 <script> 段，依然要写 export default { data() {...}, methods: {...} }。

原文依据：《初心与使命》——「我们采用的是"选项式"API，不是组合式，为保持风格统一，禁止使用"组合式"API」。
02 · 零基础背景包

读这章前先知道的 4 件事

背景①第一章学的是「无构建」写法，真实项目不是那样

第一章用 CDN（<script src="unpkg.com/vue@3...">）直接在 HTML 里写 Vue。那是无构建步骤的试用方式。真实项目（包括本项目员工端 H5）会把 Vue 写进 .vue 文件，用脚手架编译。这章就是从「试用」跨到「真实项目」。

读这章需要它：否则你不知道为什么要有 .vue 这种新文件类型。
背景②什么是「模块」（ES Module）

JS 原生支持把代码拆成一个个文件互相引用：export default { ... } 导出一个东西，import X from './x' 引入它。SFC 编译后的产物就是标准 ES 模块。所以选项式组件写 export default { data, methods }，本质就是导出一个对象。

读这章需要它：不懂 export default，就看不懂 .vue 里的 <script> 段。
背景③什么是「构建工具」

浏览器原生不认识 .vue 文件。需要有个工具把 .vue 编译成浏览器认识的 JS + CSS + HTML，这个过程叫「构建」。Vite 就是干这个的。开发时它还顺带起一个本地服务器，改代码自动刷新（热更新）。

读这章需要它：否则你会困惑「.vue 文件浏览器怎么直接打开」——它打不开，必须先构建。
背景④package.json 与 npm 脚本

每个 Node 项目根目录有个 package.json，记录依赖和脚本。npm run dev、npm run build 这些命令，本质是跑 package.json 里 scripts 字段定义的命令。npm install 则是按依赖列表把第三方库下载到 node_modules。

读这章需要它：脚手架创建项目后第一件事永远是 npm install。
03 · 这章在回答什么

表面 vs 深层问题

表层：怎么用脚手架建 Vue 项目、.vue 文件怎么写。

深层：从「HTML 里塞个 CDN」到「工程化项目」，中间多了什么？为什么官方要发明 .vue 这种格式？以及——为什么官网示例写的 <script setup>，到了本项目要换成 export default {}？

只记一个问题：.vue 文件的三段式里，<script> 段凭什么能既写 <script setup> 又写 export default {}？本项目为什么选后者？

带着这个问题读后面所有「核心判断」，尤其是判断⑥。
04 · 结构地图

从 CDN 到工程化：一条递进线

第一章（CDN/无构建）和第二章（SFC/Vite）不是两个割裂的世界，而是 Vue 官方文档「快速上手」页明确给出的两种上手方式的递进。

1方式一：无构建｜CDN 引入，HTML 里写 Vue。适合试用，第一章已学。
2方式二：SFC + 构建工具 ⭐真实项目走这条｜.vue 文件 + Vite 脚手架。
3SFC = 三段式｜<template>（视图）+ <script>（逻辑）+ <style>（样式）同处一文件。
4Vite 脚手架｜npm create vue@latest 起项目，npm run dev 开发，npm run build 打包。
5关键交叉点｜<script> 段可写「选项式」或「组合式」——本项目锁死选项式。
节点关系：SFC（文件格式）与 API 风格（选项式/组合式）是两个正交维度。SFC 决定「文件怎么组织」，API 风格决定「script 里代码怎么写」。不要混为一谈。
05 · 核心判断

6 条核心判断（每条都带原文依据）

事实① 一个 .vue 文件由三种顶层语言块构成：<template>、<script>、<style>。

这是 SFC 的官方定义。三个块分别承载视图、逻辑、样式，封装在同一个文件里。还可选加自定义块（如 <i18n>）。

原文依据：Vue 官网《SFC 语法定义》——「每一个 *.vue 文件都由三种顶层语言块构成：<template>、<script> 和 <style>。」
事实② SFC 在语法上兼容 HTML，编译后是标准 ES 模块。

这意味着浏览器不能直接打开 .vue，必须经 @vue/compiler-sfc 编译。编译产物是标准 JS（ESM）+ CSS，能被浏览器和打包工具识别。

原文依据：Vue 官网《SFC 语法定义》——「一个 Vue 单文件组件在语法上是兼容 HTML 的」「编译后的 SFC 是一个标准的 JavaScript (ES) 模块。」
事实③ Vue 官方推荐的脚手架是 create-vue，底层用 Vite。

命令是 npm create vue@latest。它专为 Vue 项目优化，提供 TypeScript / Router / Pinia / ESLint 等配置选项。与通用的 npm create vite@latest 不同。

原文依据：Vue 官网《快速上手》方式二 + Vite 官方文档——「npm create vue@latest」「Vite 是一种新型前端构建工具，能够显著提升前端开发体验。」
事实④ <template>、<script>、<style> 的顺序灵活，语义不可互换。

三个块写在上在下都行（习惯上 template/script/style 自上而下），但每段的职责固定：template 管结构、script 管逻辑、style 管样式。<style> 可有多个，<template> 和 <script> 各最多一个。

原文依据：Vue 官网《SFC 语法定义》——三段顶级语言块的数量限制与语义说明。
事实⑤ <style scoped> 让样式只作用于当前组件，不是全局。

加上 scoped 属性，Vite 编译时会给本组件的元素加一个唯一 data 属性，CSS 选择器也附上对应属性，实现「作用域隔离」。不加 scoped 则样式是全局的，可能污染别的组件。

原文依据：Vue 官网《SFC 语法定义》——<style> 支持 scoped（作用域隔离）。
判断⑥ SFC 里 <script> 段既能写 <script setup>（组合式），也能写 export default {}（选项式）——本项目锁死后者。

这是本章最重要的判断。官网大多数 SFC 示例用 <script setup>，因为 Vue3 官方在新文档里主推组合式。但 SFC 格式本身不绑定任何一种 API 风格——<script> 段写 export default { data, methods } 同样合法、同样能跑。老田的项目为了风格统一，强制选选项式。

原文依据：Vue 官网《快速上手》方式二的 SFC 示例本身用的是 <script>export default { data() {...} }</script>（选项式）+ 《初心与使命》项目约束。
06 · 关键概念拆解

6 个关键概念

① 单文件组件（SFC，Single-File Component）

定义：Vue 专属的 *.vue 文件格式，把一个组件的视图、逻辑、样式封装在一个文件里。

长这样（本项目选项式版本）：

选项式 SFC 骨架
<template>
  <button @click="count++">count is {{ count }}</button>
</template>

<script>
export default {
  data() {
    return { count: 0 }
  }
}
</script>

<style scoped>
button { font-weight: bold; }
</style>
原文依据：Vue 官网《快速上手》方式二示例。
不要误解成：SFC 必须 <script setup>。本项目用 export default {} 也是标准 SFC。

② <template> 段

定义：放组件的 HTML 结构，用第一章学的模板语法（{{ }}、v-bind/:、v-on/@、v-model）。

每个 .vue 最多一个顶层 <template>。里面写的就是你在 HTML 里见过的那套。

和第一章唯一区别：第一章写在 HTML 文件里的 <div id="app">，现在搬进了 <template>。

③ <script> 段（选项式写法）

定义：放组件的 JS 逻辑。本项目里永远是 export default { data, methods, mounted ... }。

关键：用 export default 导出一个对象，这个对象的形状就是第一章学的「选项式三件套」。SFC 编译器会把它当作组件选项处理。

不要误解成：SFC 里的 <script> 和普通 .js 文件里的 JS 不一样。它们就是同一种 JS，只是被 SFC 编译器编排进了组件。ES 模块语法（import/export）照常用。

④ <style scoped> 段

定义：放组件的 CSS。scoped 让样式只作用于本组件，不泄漏到别的组件。

原理：编译时给本组件元素和 CSS 选择器都加上 data-v-xxxxxx 属性，精确匹配。

不要误解成：scoped 是「完全隔离」。子组件的根元素仍会被父组件的 scoped 样式命中（这是官方设计的，便于布局）。它帮助理解：为什么 scoped 后改样式不污染别的页面。

⑤ Vite 与开发服务器

定义：Vite 是新型前端构建工具。npm run dev 启动后，它在本地起一个服务器（默认 localhost:5173），并提供热模块替换（HMR）——改了 .vue，浏览器只更新改动的部分，不用整页刷新。

对照：以前用 Webpack，项目大了开发服务器启动要几十秒；Vite 用原生 ESM 按需编译，启动几乎瞬时。

原文依据：Vite 官方文档——「Vite 是一种新型前端构建工具，能够显著提升前端开发体验。」

⑥ <script setup> vs export default {}（本项目选后者）

定义：两种「在 <script> 段里组织组件逻辑」的写法。

维度<script setup>（组合式）export default {}（选项式）
语法import { ref } from 'vue'export default { data(){...} }
this不用 this用 this.xxx 访问状态
官方主推✅ Vue3 新文档默认❌ 但完全支持
本项目禁止强制
原文依据：《初心与使命》——「禁止使用"组合式"API」+ Vue《快速上手》示例本身用的是 export default。

09 · 误解与边界

4 个最容易踩的坑

误解①：.vue 文件必须用 <script setup>

真相：SFC 格式不绑定 API 风格。<script> 段写 export default { data, methods }（选项式）和写 <script setup>（组合式）都合法。官网默认展示 setup 是因为主推组合式，但本项目禁组合式。

原文依据：Vue《快速上手》示例本身用的就是 export default + 《初心与使命》项目约束。

误解②：.vue 能直接在浏览器打开

真相：浏览器不认 .vue。必须经 @vue/compiler-sfc 编译成标准 JS（ESM）+ CSS。所以必须有构建步骤（Vite），不像第一章的 CDN 方式可以双击 HTML 打开。

原文依据：《SFC 语法定义》——「编译后的 SFC 是一个标准的 JavaScript (ES) 模块。」

误解③：scoped = 完全样式隔离

真相：scoped 通过 data-v-xxxxxx 属性匹配实现隔离，但子组件的根元素仍会被父组件 scoped 样式命中（这是官方设计，方便布局）。不是 100% 隔离。

误解④：Vite = Webpack 的换皮

真相：Vite 和 Webpack 架构不同。Webpack 是「先全量打包再启动」，Vite 是「开发时用原生 ESM 按需编译、生产时再 Rollup 打包」。开发服务器启动速度差几个量级，不要用 Webpack 的心智模型套 Vite。

边界：本章只覆盖「SFC 三段式 + Vite 脚手架起项目 + 选项式 vs 组合式在 SFC 里的取舍」。<style> 的 CSS Modules、预处理器（Sass/Less）、Vite 高级配置（代理/别名/插件）、Vue Router/Pinia 都不在本章。本章目标是「能起项目、能读 .vue、不踩 API 风格坑」。
10 · 背景知识缺口

要完全吃透还差什么

现在就该补（阻碍你起项目）：

  • Node.js 与 npm 基础—— 装好 Node，会 npm install / npm run dev。起项目第一步。
  • ES6 模块（export default / import）—— 选项式组件就是 export default 导出的对象。

可以晚点补（不影响入门）：

  • Vite 配置（vite.config.js）—— 代理、别名、环境变量。本项目要请求后端接口时会用到代理。
  • 预处理器（Sass/Less）—— <style lang="scss">。看员工端项目用不用再学。
  • Vue Router / Pinia—— 路由和状态管理。脚手架可选装，第四章组件化之后再碰。
下一步追问：「员工端 H5 项目到底用的什么脚手架配置？启没启 TS、Router、Pinia？」—— 这要问土豆，参考员工端现有项目。本章给的是通用骨架，具体配置以员工端为准。

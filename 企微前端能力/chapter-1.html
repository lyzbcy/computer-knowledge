01 · 学习目标

学完这一章你要能做到

  • 说清前端能力 JSAPI 的范畴：这是企微「网页开发工具包」提供的通用前端能力（选人/转发/定位/扫码/界面），区别于第五章讲的鉴权链路与数据专区接口。
  • 会用 selectEnterpriseContact 选人：能写 from/mode/type 参数，知道返回的 arrayOfSelector 含 member/department/tag。
  • 会用 shareAppMessage 转发：能传 title/desc/link/imgUrl，知道它转发到企微会话（区别于 sendChatMessage）。
  • 知道 getLocation / scanQRCode 的用法：type 决定坐标系，needResult 决定扫码结果给谁。
  • 记住一个铁律：所有这些能力调用前提都是 ww.register 成功（呼应第五章），否则报 invalid signature / permission denied。
阅读路径：背景包 → 结构地图 → 核心判断 → 关键概念 → 做关卡题 → 生成报告。
本章定位

这是企微前端能力系列的通用能力章。第五章你已经学了鉴权（ww.register）和数据专区接口（selectDatazoneDoc/File）；本章回答：「除了选文档选文件，企微 H5 还能用哪些原生能力，让应用像微信 JS-SDK 一样 native 化」。

原文依据：企微 JS-SDK 概述——「借助企业微信高效地使用拍照、选图、语音、位置等手机系统的能力，同时可以直接使用企业微信分享、扫一扫等企业微信特有的能力」。
02 · 零基础背景包

读这章前先知道的 4 件事

背景①第五章学了鉴权和专区接口，但企微前端能力还有一大类通用接口

第五章的 ww.register（鉴权）+ selectDatazoneDoc/File（专区接口）只是企微 JS-SDK 的一部分。还有一大类通用前端能力接口：选人（selectEnterpriseContact）、转发（shareAppMessage）、定位（getLocation）、扫码（scanQRCode）、界面（closeWindow 等）。本章就是讲这一类。

读这章需要它：知道企微前端能力不止专区接口，还有一套像微信 JS-SDK 的通用能力。
背景②这些能力和第五章的专区接口共享同一个调用前提：ww.register

不管是 selectEnterpriseContact、shareAppMessage 还是 getLocation，调用前提都一样——必须先 ww.register 注册身份成功（拿到合法签名）。这就是第五章那个铁律在本章的延续：register 失败，所有 ww.* 都调不了。

读这章需要它：本章不再重复讲鉴权细节，但每个能力的前提都是"register 成功"。
背景③旧版用 wx.invoke('xxx')，新版用 ww.xxx()，建议用新版

企微 JS-SDK 有两套写法：旧版 jweixin 是 wx.invoke('接口名', 参数, 回调)，靠 res.err_msg === 'xxx:ok' 判断；新版 wecom-jssdk 是 ww.接口名(参数)，直接用 success 回调或 Promise。官方推荐新版，本章以新版为主。

读这章需要它：看到代码里 ww.xxx() 是新版，看到 wx.invoke(...) 是旧版，别搞混。
背景④这些能力让 H5 应用能调用企微客户端的原生功能

企业微信 JS-SDK 是企微面向网页开发者提供的开发工具包。用了它，你的 H5 网页就能用手机系统的能力（定位、扫一扫），还能用企微特有的能力（选企业通讯录成员、转发到企微会话）——就像微信网页用的微信 JS-SDK 一样，让你的网页在企微里"native 化"。

读这章需要它：理解为什么要在 H5 里调一套"企微专属 API"——为了用上客户端原生能力。
03 · 这章在回答什么

表层 vs 深层问题

表层：企微 H5 还有哪些前端能力可用？选人、转发、定位、扫码、界面分别怎么调？

深层：这些能力怎么和第五章的鉴权链路配合，让 H5 应用真正"native 化"——从一个只能显示内容的网页，变成能选人、能转发、能定位、能扫码的"准原生应用"？

只记一个问题：在企微 H5 里，调一个通用前端能力（比如扫码），从注册身份到拿到结果，骨架是什么？每个能力的"关键参数 + 返回"长什么样？

带着这个问题读「04 结构地图」，那个"通用调用骨架"就是答案。
04 · 结构地图

五大前端能力，都建立在 ww.register 之上

企微前端能力分五大类，全部建立在第五章学的 ww.register 之上。下面这张地图把五类能力和它们的代表接口列清楚，每类都是"register 成功 → 调 ww.xxx() → 回调拿结果"。

0身份注册 ⭐｜ww.register 成功（corpId + agentId + 签名回调）—— 这是下面五类能力的共同前提。
1通讯录 · 选人｜ww.selectEnterpriseContact——选企业成员/部门/标签，企微特有（微信没有）。
2分享 · 转发｜ww.shareAppMessage——自定义转发到企微会话，支持 text/news/textcard 等消息。
3位置 · 定位｜ww.getLocation——获取经纬度，type 参数决定坐标系（wgs84/gcj02）。
4系统界面 · 扫码｜ww.scanQRCode——调起扫一扫，needResult 决定结果给企微还是返回页面。
5界面控制｜ww.closeWindow / hideOptionMenu / hideMenuItems 等——控制窗口、菜单、返回。
节点关系：步骤 0（ww.register）是五类能力的共同根节点——不 register 直接调任何能力都会报无权限。这条因果线和第五章完全一致，本章是它的延伸。
05 · 核心判断

6 条核心判断（每条都带原文依据）

事实① 所有前端能力 JSAPI 调用前提都是 ww.register 成功（呼应第五章）。

这是贯穿全章的铁律。不管选人、转发、定位还是扫码，每个接口的文档都写了"调用前提：ww.register 成功"。register 失败就报 invalid signature / permission denied。这和第五章的专区接口一模一样。

原文依据：企微 JS-SDK 概述（90513）——「以下每个接口调用前，必须先 ww.register 注册成功，否则报 invalid signature / permission denied」。
事实② selectEnterpriseContact 可选成员/部门/标签，是企微特有能力（微信没有）。

这个接口选的是企业通讯录里的成员、部门、标签，由 type 参数控制可选范围。微信生态里没有"选企业部门/标签"这种概念，所以它是企微独有的能力，适合做"指派给某人/某部门"这类企业场景。

原文依据：企微《选择通讯录成员》（91821）——「选择企业通讯录中的成员/部门/标签」，type 可选 member/department/tag。
事实③ shareAppMessage 转发到企微会话，支持 text/news/textcard 等消息类型。

shareAppMessage 用于自定义转发到企微会话：用户点右上角"转发"时，用你指定的 title/desc/link/imgUrl，而不是默认抓取页面。它和企业微信 2.4.5 起支持。注意它和侧边栏"发到当前会话"的 ww.sendChatMessage 是两个不同场景。

原文依据：企微《自定义转发到会话》（100505）——「兼容性：企业微信 2.4.5 起支持」，参数 title/desc/link/imgUrl。
事实④ getLocation 获取经纬度，type 参数决定坐标系（wgs84/gcj02）。

getLocation 返回 latitude/longitude/speed/accuracy。关键是 type：'wgs84' 是 GPS 坐标（默认），'gcj02' 是火星坐标（国内地图常用）。选错坐标系会让定位偏几十到几百米。还要注意企微是 ww.getLocation，别和公众号/小程序的 wx.getLocation 混淆。

原文依据：企微《获取地理位置》（100578）——「type：'wgs84'（默认 GPS 坐标）/ 'gcj02'（火星坐标）」，返回 latitude/longitude/accuracy。
事实⑤ scanQRCode 扫一扫，needResult 决定结果由企微处理还是返回给页面。

scanQRCode 的 needResult：0（默认）扫描结果由企业微信处理（比如扫到网址直接打开），1 则把扫描结果字符串 res.resultStr 直接返回给你的页面。scanType 还能指定 ['qrCode','barCode']（二维码/一维码）。

原文依据：企微《调起扫一扫》（90492）——「needResult：0 扫描结果由企业微信处理；1 直接返回扫描结果」。
判断⑥ 旧版 wx.invoke 和新版 ww.xxx 都能用，官方推荐新版。

同一个能力有两套写法：旧版 wx.invoke('selectEnterpriseContact', 参数, 回调) 靠 res.err_msg==='xxx:ok' 判断；新版 ww.selectEnterpriseContact({ ...success(res){} }) 直接用 success 回调。两者都能跑，但官方主推新版 wecom-jssdk，新项目建议直接用 ww.*。

原文依据：企微各接口文档均给出"新版示例 + 旧版 jweixin 调用"两段对照；概述（90513）主推 @wecom/jssdk。
06 · 关键概念拆解

6 个关键概念

① selectEnterpriseContact（选人）

定义：选择企业通讯录中的成员/部门/标签，企微特有能力。返回 res.result.arrayOfSelector，每项含 type（member/department/tag）和 id。

关键参数说明
from选人来源，'single' 选单人、'multiple' 选多人
mode'single'（单选）/ 'multi'（多选）
type可选项数组，如 ['member','department','tag']
selectedUserIds / selectedDepartmentIds默认选中项
ww.selectEnterpriseContact({
  from: 'multiple',
  mode: 'multi',
  type: ['member', 'department'],
  selectedUserIds: ['zhangsan'],
  success(res) {
    console.log(res.result.arrayOfSelector);
  },
});
原文依据：《选择通讯录成员》（91821）。
不要误解成：它能选微信好友。它选的是企业通讯录，范围由 type 限定。

② shareAppMessage（转发）

定义：自定义转发到企微会话。用户点右上角"转发"时，用你指定的内容（而不是企微默认抓的）。

属性必填说明
title是分享标题
desc是分享描述
link是分享链接
imgUrl是分享图标
enableIdTrans否4.0.20 起，1 开启 id 转译，仅供第三方应用
ww.shareAppMessage({
  title: '企业微信',
  desc: '让每个企业都有自己的微信',
  link: 'https://work.weixin.qq.com/',
  imgUrl: 'https://res.mail.qq.com/.../index_share_logo.png',
});
不要误解成：shareAppMessage 就是"发到当前会话"。"发到当前会话"要用侧边栏的 ww.sendChatMessage，shareAppMessage 是"转发到任意企微会话"。

③ getLocation（定位）

定义：获取当前地理位置（经纬度）。调用前提除了 register 成功，应用还要有定位权限，否则报 getLocation:nopermission。

参数/返回说明
type'wgs84'（默认 GPS 坐标）/ 'gcj02'（火星坐标）
latitude / longitude经纬度
accuracy精度
speed速度
ww.getLocation({
  type: 'wgs84',
  success(res) {
    console.log(res.latitude, res.longitude, res.accuracy);
  },
});
不要混淆：企微 JS-SDK 是 ww.getLocation，不是公众号的 wx.getLocation。

④ scanQRCode（扫码）

定义：调起企微扫一扫。needResult 决定结果给谁，scanType 决定扫什么码。

参数说明
needResult0（默认，结果由企微处理）/ 1（结果返回给页面 res.resultStr）
scanType['qrCode','barCode']，二维码 / 一维码
wx.scanQRCode({
  needResult: 0,                    // 0 结果由企微处理；1 直接返回结果
  scanType: ['qrCode', 'barCode'],  // 二维码 / 一维码
  success: function (res) {
    var result = res.resultStr;     // needResult=1 时的扫码结果
  },
});
原文依据：《调起扫一扫》（90492）官方原示例。

⑤ 界面能力（setTitleBarColor / closeWindow 等）

定义：JS-SDK「界面」分类下的一组前端界面控制能力，均需 register 成功后调用。用来控制窗口、菜单、返回行为。

接口作用
ww.closeWindow()关闭当前窗口
ww.hideOptionMenu() / showOptionMenu()隐藏 / 显示右上角菜单
ww.hideMenuItems({menuList}) / showMenuItems()批量隐藏 / 显示功能按钮
ww.hideAllNonBaseMenuItem() / showAllNonBaseMenuItem()隐藏 / 显示非基础按钮
监听页面返回 / 打开默认浏览器 / 监听截屏其它界面类能力
原文依据：企微 JS-SDK「界面」分类（90513 目录）——关闭窗口/隐藏菜单/监听返回/打开默认浏览器/监听截屏等。

⑥ 通用调用骨架：register 成功 → 调 ww.xxx() → 回调拿结果

定义：所有前端能力共用同一条调用链。第一步 register（服务端给签名），第二步在 .then() 里调任意 ww.* 能力，第三步在 success 回调拿结果。这是本章和第五章共通的"骨架"。

register 成功后调任意能力
import ww from '@wecom/jssdk';

// 1. 服务端用 access_token + jsapi_ticket 生成签名，前端拿到后注册
ww.register({
  corpId: 'wwxxxxxxxxxxxxxxx',
  agentId: 1000002,
  jsApiList: ['selectEnterpriseContact', 'shareAppMessage',
              'getLocation', 'scanQRCode', 'closeWindow'],
  getConfigSignature() {
    return fetch('/api/wecom/jssign?url=' +
      encodeURIComponent(location.href.split('#')[0]))
      .then(r => r.json()); // { timestamp, noncestr, signature }
  },
}).then(() => {
  // 2. 注册成功后才能调用任意 ww.* 接口
  ww.scanQRCode({ needResult: 1, scanType: ['qrCode'] });
}).catch(err => console.error('register fail', err));
不要误解成：能不 register 直接调 ww.scanQRCode。必须在 register 的 .then() 里、或确认 register 成功后再调，否则报无权限。

09 · 误解与边界

4 个最容易踩的坑

误解①：不 register 也能调前端能力

真相：所有前端能力 JSAPI（选人/转发/定位/扫码/界面）的调用前提都是 ww.register 成功。这是和第五章完全一致的铁律。不 register 直接调，报 invalid signature / permission denied。

原文依据：概述（90513）——「每个接口调用前，必须先 ww.register 注册成功」。

误解②：shareAppMessage 就是"发到当前会话"

真相：shareAppMessage 是自定义转发到企微会话（用户点右上角"转发"）。而聊天工具栏侧边栏场景下"发到当前会话"要用 ww.sendChatMessage(msgType)。两者场景和接口都不同，别混用。

原文依据：《自定义转发到会话》（100505）末尾区分说明。

误解③：ww.getLocation 和 wx.getLocation 一样

真相：企微 JS-SDK 是 ww.getLocation，微信公众号/小程序是 wx.getLocation。两者 SDK 不同、注册方式不同。企微项目里要用 ww.* 系列，别照搬公众号代码。另外 type 选错坐标系会让定位偏几十到几百米。

原文依据：《获取地理位置》（100578）——「勿与微信公众号/小程序的 wx.getLocation 混淆」。

误解④：界面能力（closeWindow 等）随便调

真相：界面能力（closeWindow/hideOptionMenu/hideMenuItems 等）虽然简单，但同样需要 register 成功后才能调用。另外导航栏标题颜色等在新版 SDK 中随主题统一管控，旧版 wx.setNavigationBarTitle 类接口以官方"界面"目录为准。

原文依据：企微 JS-SDK「界面」分类——「均需 ww.register 成功后调用」。
边界：本章覆盖「选人 + 转发 + 定位 + 扫码 + 界面」五大通用前端能力及通用调用骨架。后端签名算法细节、可信域名备案流程、各能力在企业内的具体权限开通、图像/音频/网络/剪贴板等其它分类接口，属于延伸内容，需查官方文档或问同事。
10 · 背景知识缺口

要完全落地还差什么

必须补（查官方文档 / 找同事确认）：

  • 各能力的 jsApiList 注册—— register 时 jsApiList 要把用到的接口名都列上（selectEnterpriseContact/shareAppMessage/getLocation/scanQRCode/closeWindow 等），对照官方文档逐个核对接口名拼写。
  • 可信域名与权限开通—— 调用前提除了 register，部分能力（如 getLocation）要求应用已开通对应权限，否则报 nopermission。找居易/老田确认应用权限。
  • 定位坐标系选择—— wgs84 还是 gcj02 取决于你下游用什么地图（高德/腾讯用 gcj02，GPS 原始用 wgs84）。看业务对接的地图 SDK。

可以晚点补：

  • 图像/音频/网络/剪贴板等其它分类—— 本章只讲五大类，JS-SDK 还有选图、录音、网络状态、剪贴板等，需要时查概述（90513）目录。
  • 旧版 wx.invoke 全套迁移—— 若维护老项目遇到 wx.invoke 写法，对照各接口文档的"旧版 jweixin 调用"段落即可，新项目直接用 ww.*。

后端学习资料

Java 17 + Spring Boot 后端三阶段入门到进阶 · 一人开发 学习资料

后端学习资料

适用对象：新入职的应届毕业生，以及需要补齐 Java 后端基础的同学

技术基线

Java 17、Maven、Spring Boot、Spring Cloud、Nacos、OpenFeign、MyBatis/MyBatis-Plus、PostgreSQL、Redis、RocketMQ/Spring Cloud Stream

1. 学习阶段

学习分为三个阶段。完成每个阶段的验收作品和检查项后，再进入下一阶段。

阶段能力目标能完成的典型工作暂不要求独立完成
第一阶段：开发入门从零基础到能读懂项目主链路，并完成简单单服务功能简单 CRUD、详情、列表、分页、校验、Mapper/XML SQL、日志、基础测试复杂跨服务编排、缓存一致性、MQ 可靠性、分片设计
第二阶段：独立开发能独立交付项目中大多数常见业务功能复杂查询、本地事务、Feign/Nacos、Redis、MQ 收发与消费、定时任务、集成测试、常见问题排查跨资源最终一致性、复杂并发、平台级治理和疑难线上问题
第三阶段：工程进阶能负责复杂分布式功能及生产质量闭环事务消息、重试/死信与补偿、分库分表、多租户机制、并发、网关安全、可观测性、性能和容量治理新业务领域和服务独有第三方 SDK 仍需按需学习

2. 项目技术版本

学习和练习使用以下版本。进入具体服务后，以该服务的 effective POM 为准。

组件项目版本学习资料选择

3. 第一阶段：开发入门

阶段目标

完成本阶段后，学习者应该能够：

  • 看懂 Controller -> Service -> Mapper -> PostgreSQL 主链路。
  • 在已有服务中完成低风险的简单 CRUD、详情和分页需求。
  • 写出带租户条件、参数绑定和明确字段的安全 SQL。
  • 正确使用项目统一结果、校验、异常和日志组件。
  • 为正常与异常路径编写基础测试。

I1. 开发环境、Git、HTTP 与 JSON

学习内容：

  • 终端、目录、进程、端口、环境变量和日志文件。
  • Git 的 clone、branch、commit、diff、merge/rebase 基本概念。
  • HTTP 请求/响应、方法、状态码、Header、JSON、Cookie、Bearer Token。
  • DNS、IP、端口、客户端/服务端、连接超时和读取超时。

验收：

  • 能创建分支、提交修改、查看 diff 并处理简单冲突。
  • 能用 curl 发送 GET/POST JSON 请求，解释 400、401、403、404、409、500。
  • 能区分“端口未监听、连接超时、HTTP 500”分别发生在哪一层。

I2. Java 17 基础

必须掌握：

  • 变量、类型、分支、循环、方法、包和访问控制。
  • 类、接口、继承、组合、多态和不可变对象。
  • String、包装类型、BigDecimal、枚举、日期时间 API。
  • 数组、List、Set、Map，泛型，equals/hashCode。
  • 异常分类、抛出/捕获、保留 cause、try-with-resources。
  • Lambda、方法引用、Stream 的 map/filter/collect/groupingBy/toMap。
  • 空集合、null 和 Optional 的使用边界。
  • Java 9～17 常用特性：局部变量 var、switch 表达式、文本块、record、instanceof 模式匹配。

暂缓：GUI、JNI、模块系统深入、虚拟线程、复杂 JVM 调优和响应式编程。 练习：写一个“客户跟进记录”命令行程序，支持新增、修改、查询、按员工分组统计，并用测试覆盖空值、重复 ID、非法状态和金额比较。

I3. Maven 与多模块基础

学习内容：

  • POM 坐标、依赖、scope、传递依赖、排除和插件。
  • compile / test / package / install / deploy 生命周期。
  • parent、dependencyManagement、BOM、properties、profile。
  • 多模块 reactor、-pl、-am、Maven Wrapper、私服和 settings.xml。
  • 使用 dependency:tree 与 help:effective-pom 排查版本。

验收：能把 Java 练习拆成多个 Maven 模块，并解释“依赖声明了但类找不到”和“版本未生效”的常见原因。

I4. SQL 与 PostgreSQL 基础

这一节必须在 MyBatis 之前完成。 学习内容：

  • 表、行、列、主键、唯一约束、非空约束和默认值。
  • SELECT / INSERT / UPDATE / DELETE，WHERE、ORDER BY、LIMIT。
  • INNER/LEFT JOIN、聚合、GROUP BY、HAVING、子查询和简单 CTE。
  • 事务和 ACID 的直观含义。
  • PostgreSQL 常用类型：bigint、numeric、text/varchar、boolean、timestamp、jsonb。
  • B-tree 索引、组合索引和唯一索引的基础场景。

项目规则：

  • SELECT 和 INSERT 明确列名，不使用 SELECT *。
  • UPDATE/DELETE 必须有安全条件和租户条件。
  • 值使用参数绑定；动态字段、表名和排序字段只能来自服务端白名单。
  • 新租户表包含项目要求的主键、时间、逻辑删除、版本和 ws_tenant_id。

验收：为客户、员工和跟进记录设计表、唯一约束和索引，写出 CRUD、分页和统计 SQL。

I5. Spring 核心与 Spring Boot

学习内容：

  • IoC/DI、Bean、组件扫描、配置类和构造器注入。
  • AOP/代理的直观概念，知道事务、异步和校验为何可能受自调用影响。
  • @SpringBootApplication、Starter、自动配置和条件装配。
  • application.yaml、profile、环境变量和 @ConfigurationProperties。
  • Spring Boot 3 的 Jakarta 包名；启动日志、Bean 冲突和循环依赖识别。
  • 只需先读懂 Nacos spring.config.import、namespace 和 profile 如何影响服务启动，服务发现细节放到第二阶段。

I6. REST API、校验、异常和日志

学习内容：

  • Controller、@RequestMapping、@RequestBody 和 JSON 序列化。
  • Request/Response/DTO/DO 的职责，不直接暴露 DO。
  • Jakarta Validation 的非空、长度、范围、集合大小和级联校验。
  • 全局异常处理、业务异常、未知异常和统一 Result<T>。
  • SLF4J 占位符、异常对象、日志级别和敏感数据保护。

项目约定：

  • Controller 只做协议入口、校验、鉴权编排、转换和调用 Service。
  • 领域接口使用 /feign，BFF 使用 /bff，延续所在服务的版本与 POST 约定。
  • 使用 Result<T>、ResultUtils、统一异常和 @WebLog 等组件。
  • 登录身份、租户和资源归属由服务端可信上下文确认，不能信任客户端角色声明。

I7. MyBatis / MyBatis-Plus 入门

在已经会手写 SQL 后学习：

  • Mapper 接口、XML namespace、#{} 参数绑定和 resultMap。
  • BaseMapper、@TableName/@TableId/@TableField/@TableLogic。
  • LambdaQueryWrapper、Wrappers 和基础 IPage/Page 分页。
  • 简单 CRUD 使用 MyBatis-Plus；需要 JOIN、统计或复杂映射时使用明确的 Mapper XML。
  • Wrapper 每次请求新建，不能跨线程复用，也不能通过 RPC 传输。

I8. 简单分层和基础测试

入门阶段只要求先掌握： 测试从本阶段开始：

  • Java 逻辑使用 TestNG 单元测试。
  • Service 通过构造器注入替身 Repository/Mapper 测试。
  • Mapper/SQL 使用隔离数据库做基础集成测试。
  • 每个功能至少覆盖成功、空值、非法参数、无数据、重复数据和数据库异常。

第一阶段验收作品

  • 在项目模板中新增“跟进记录”功能：建表、DO、Mapper/XML、Service、Request/Response 和 Controller。
  • 支持新增、修改、详情、逻辑删除和条件分页。
  • 所有租户查询与修改携带正确的 wsTenantId。
  • 有完整参数校验、业务错误、日志和不少于 15 个测试场景。
  • Maven 构建通过，能够讲清一次请求从入口到 SQL 的完整路径。

4. 第二阶段：独立开发

阶段目标

完成本阶段后，学习者应该能够独立交付大多数日常需求，包括跨服务调用、缓存、消息、事务和任务，并能排查常见故障。

D1. 项目模块化与代码阅读

在掌握简单分层后，再理解现代服务的模块： 必须掌握：

  • Domain 不依赖 Web、Feign 或具体中间件。
  • Application 不绕过已有 Repository 直接访问 Mapper。
  • 外部调用集中在 Dependency/Adapter/Client 防腐层。
  • Request、DTO、Entity、DO、Response 只在相邻层显式转换。
  • 不使用 BeanUtils.copyProperties 隐藏对象转换。

D2. PostgreSQL 与 MyBatis 进阶

学习内容：

  • 窗口函数、复杂聚合、CTE、INSERT ... ON CONFLICT 和批量写入。
  • jsonb 查询与 GIN 索引、数组、时间和游标分页。
  • MyBatis 动态 SQL：if / choose / foreach / where / set。
  • MyBatis-Plus Lambda Query/Update Wrapper、自定义 Mapper 分页和插件链。
  • EXPLAIN (ANALYZE, BUFFERS)、慢 SQL 和索引选择基础。
  • 使用已有分片表时必须携带分片路由字段，能够识别不带路由字段的全路由风险。

D3. 本地事务与并发正确性

需要理解 @Transactional 的 AOP、传播和回滚原理，但本项目业务代码按规范使用 TransactionTemplate，禁止新增 @Transactional。 必须掌握：

  • 用 TransactionTemplate 明确事务结果、异常和回滚边界。
  • 事务只覆盖必须原子完成的本地数据库操作。
  • Feign、文件、MQ、sleep 和长循环不放在数据库事务中。
  • 唯一约束、版本字段、幂等键和原子状态迁移的基本使用。
  • 知道数据库事务不能回滚 Redis、MQ、ES 或远程服务。

D4. Nacos、OpenFeign 与服务调用

学习内容：

  • Nacos 的 namespace、group、Data ID、服务实例和健康状态。
  • spring.config.import=nacos:...、配置覆盖、环境隔离和配置刷新。
  • 服务注册与发现、Spring Cloud LoadBalancer。
  • @FeignClient、服务名、contextId、Request/Response 契约。
  • Feign Controller 实现共享接口的项目模式。
  • 连接/读取超时、总体时间预算、错误码校验和日志。
  • 避免循环内逐条 Feign，优先使用批量接口。
  • 已发布 Feign、DTO 和枚举只扩展，不破坏旧契约。

验收：能排查未注册、namespace 不一致、配置未加载、404、超时、序列化错误和下游业务失败。

D5. Redis 常规开发

学习内容：

  • String、Hash、List、Set、Sorted Set 的场景和复杂度。
  • 项目统一 Redis Client/RedisTemplate/Redisson 的使用边界和序列化。
  • Key 命名、TTL、随机抖动、空值缓存和容量意识。
  • Cache Aside、更新/删除顺序、不一致窗口和失败补偿。
  • 原子命令、Lua、计数器、简单限流、SCAN。
  • 缓存穿透、击穿、雪崩、Big Key 和 Hot Key。

项目约定：默认设置 TTL；线上禁止 KEYS/FLUSHALL/FLUSHDB；不绕过框架自建 Redis 客户端。

D6. RocketMQ / Spring Cloud Stream 常规开发

学习内容：

  • Topic、Message、Tag、Key、Producer、Consumer、Consumer Group 和 Offset。
  • 普通消息与顺序消息的基本场景；同步/异步发送。
  • Spring Cloud Stream 的 Binder、binding、destination、group 和 content-type。
  • 函数式 Consumer<Message<...>> 与 StreamBridge。
  • 读懂并使用项目 MessageConsumer、Wrapper 和 RocketMQClient。
  • 消息契约包含 messageId、事件类型、版本、时间、租户、业务主键和必要负载。
  • 默认消息可能重复；消费者至少使用 messageId 或业务唯一键实现幂等。
  • 消费异常不能吞掉；可重试错误必须正确触发重试。

本阶段掌握普通消息、顺序消息及基础幂等。事务消息和复杂死信恢复在第三阶段学习。

D7. 定时任务、批处理和异步

学习内容：

  • TaskExecutor/TaskScheduler、@Async/@Scheduled 的代理限制。
  • 项目 Scheduler Starter/任务框架的使用方式。
  • 分页批处理、稳定主键游标、断点续跑、幂等、重试上限和失败终态。
  • 线程池 Bean 的核心数、最大数、有界队列、线程名、拒绝策略和优雅关闭。
  • CompletableFuture 的组合、超时、异常和指定线程池。

D8. 集成测试、调试和基础可观测性

  • TestNG @DataProvider、项目 BaseTest、MockListener/测试 Starter。
  • Service 单测、Feign 防腐层测试、Mapper SQL 测试和 Spring Boot 集成测试。
  • PostgreSQL、Redis 和 MQ 使用可重复的隔离环境进行测试。
  • 日志 correlation/trace ID，Actuator health 和 metrics 基础。
  • 能从日志、SQL 计划、依赖树和线程池状态建立故障定位链。

第二阶段验收作品

  • 在第一阶段作品上增加一个跨服务 Feign 调用、Nacos 配置和 Redis 缓存。
  • 增加“跟进记录变更”MQ 生产与消费，包含消息 ID、版本、租户和幂等处理。
  • 用 TransactionTemplate 完成“写主记录 + 写明细 + 状态变更”，并验证异常回滚。
  • 增加可续跑、限量和幂等的批处理任务。
  • 覆盖下游超时、重复提交、MQ 重复、缓存失效、并发更新和慢 SQL 测试。
  • 能独立判断接口慢来自 SQL、Redis、Feign、线程池、MQ 积压还是下游。

5. 第三阶段：工程进阶

阶段目标

完成本阶段后，学习者能够针对失败、重复、并发、扩容、安全和发布设计可验证的工程方案。

A1. MQ 可靠性进阶

  • 普通、顺序、延迟和事务消息的选择。
  • 发送结果、发送重试、重复投递、乱序、积压、消费超时和死信。
  • 顺序键与分区、消费并发度和失败阻塞的关系。
  • 可重试与不可重试错误分类；重试退避、死信告警、人工恢复和自动补偿。
  • 幂等记录与业务更新放在同一本地事务中。
  • 消息契约演进、兼容消费者和灰度发布。

A2. 事务消息与补偿一致性

  • 读懂项目中的 sendMessageInTransaction、本地事务监听与事务回查。
  • 本地事务、事务消息、幂等消费者、业务状态流转、发送记录和补偿任务的配合。
  • 处理“数据库成功、MQ 失败”“MQ 成功、消费失败”“Feign 超时但下游已成功”。
  • 关键消息可追踪、可补偿、可对账，不能捕获发送失败后返回成功。
  • 项目禁止 Seata，不假设本地事务可以回滚 Redis、MQ、ES 或第三方系统。
  • 高价值接口使用幂等键、唯一约束、频控、防重放和审计。

A3. PostgreSQL 性能、锁与数据演进

  • MVCC、Read Committed、行锁、死锁、乐观锁和锁范围。
  • 深入执行计划、统计信息、索引失效、连接池和慢 SQL 治理。
  • B-tree、GIN、唯一、组合、部分和表达式索引的选择。
  • 生产索引使用 CREATE/DROP INDEX CONCURRENTLY。
  • 新增列、双写、回填、切读和清理的兼容性 DDL 演进。

A4. ShardingSphere 与多租户机制

  • 逻辑表、真实表、数据节点、分片键、算法、路由、改写、执行和归并。
  • ShardingSphereDriver 与 jdbc:shardingsphere:classpath:... 配置路径。
  • 单路由、多路由、范围路由和全路由；跨分片 JOIN/聚合/分页成本。
  • 分片算法设计、容量预估、扩容、数据迁移和校验。
  • MyBatis-Plus TenantLine/自定义拦截器的作用、边界和旁路风险。
  • 所有租户访问携带 wsTenantId，分片表同时携带路由字段。

A5. 并发、线程池和分布式锁

  • Java 内存模型、happens-before、volatile、锁、原子类和并发集合。
  • ThreadPoolExecutor 参数、背压、队列容量、拒绝策略和上下文传递。
  • 租户、用户和 MDC/trace 上下文的 TTL 传递与 ThreadLocal.remove()。
  • 优先使用唯一约束、乐观锁和原子状态迁移，锁只保护最小范围。
  • 分布式锁明确资源、粒度、等待、租约、续期、持有者校验和失败策略。

A6. 服务容错与流量治理

  • 连接、读取和总体时间预算的分配。
  • 重试只用于幂等操作或有幂等键保护的写操作。
  • Spring Retry：@Retryable、@Backoff、重试异常分类、指数退避和随机抖动。
  • Sentinel：流量控制、熔断降级和 Gateway 适配。
  • 识别 Feign、Spring Retry 与 MQ 各层叠加造成的重试放大。
  • 模糊成功处理，避免伪造空数据作为成功降级。
  • N+1 Feign、批量接口和上下游容量匹配。

A7. 网关、安全和接口防护

  • Spring Cloud Gateway 的 route、predicate、filter、全局过滤器和限流。
  • Spring Security/OAuth2 Resource Server、JWT 校验、认证与授权。
  • 租户与资源归属校验、最小权限和服务间信任边界。
  • SQL/SpEL/模板/命令注入、文件上传、SSRF、反序列化和敏感日志。
  • 已发布 API、Feign、DTO、枚举和消息契约的兼容演进。

A8. 可观测性、性能和生产治理

  • 日志、指标、链路三支柱；trace 跨线程、Feign 和 MQ 传播。
  • Actuator health/metrics、JVM、线程池、连接池、Redis 和 MQ 积压指标。
  • JVM 内存、GC、线程 dump、CPU、锁竞争和内存泄漏基础排查。
  • 容量估算、告警、限流、优雅停机和发布检查。
  • 故障演练：下游超时、Redis 不可用、MQ 重复、数据库死锁和错误配置。

A9. 服务专项选修

在完成共同技术基线后，再按目标服务选择：

  • 客户/搜索类：Elasticsearch、索引一致性、重建和双写。
  • 导入导出类：EasyExcel/POI、流式处理、内存控制和异步导出。
  • 素材/文件类：对象存储、上传安全、回调和生命周期。
  • AI/会话类：SSE、异步编排、模型 API 和检索组件。

第三阶段验收作品

  • 将第二阶段 MQ 链路升级为项目已有的事务消息或“发送记录 + 补偿任务”方案，补齐有限重试、死信、补偿、告警和对账。
  • 在分片表上证明带与不带路由键的执行差异，并覆盖租户隔离测试。
  • 完成并发重复请求、MQ 重复、Feign 模糊成功、Redis 锁超时和数据库死锁测试。
  • 提供契约兼容、容量估算、监控指标、灰度发布和回滚清单。
  • 能在代码评审中识别事务边界、吞异常、N+1 Feign、无界线程池、全路由和缓存不一致风险。

6. 项目代码阅读顺序

不要从最大的服务开始逐文件阅读。每次围绕一条功能链，只读请求入口到数据或消息出口。

  • docs/后端开发规范.md：先了解项目允许和禁止什么。
  • 第一阶段阅读 wshoto-custom-model 的简单接口和 CRUD 链路，但暂不深究全部 DDD 概念。
  • 第二阶段完整阅读 wshoto-crm-follow 的 Feign、Controller、Application、Repository/Mapper 和 MQ Consumer。
  • 第二阶段阅读 wshoto-biz-content 的多模块分层、Nacos、ShardingSphere 驱动和消息绑定。
  • 第三阶段阅读 wshoto-customer 的多租户拦截器、复杂 Feign、大量消息绑定和历史分层。
  • 第三阶段阅读 wshoto-gateway 的 WebFlux Gateway、认证、路由和过滤器。

单条链路阅读顺序： 每读完一条链路，输出调用图、对象转换表、事务边界和失败场景表。

9. 学习方法

每个主题都遵循同一循环：

  • 阅读官方文档的概念、最小示例和限制。
  • 在独立小项目复现成功路径。
  • 主动制造一个失败：超时、重复、空值、回滚、死锁或消息重投。
  • 在 Wshoto 代码中找到同类实现，画出调用与数据流。
  • 按《后端开发规范》重写或补齐练习。
  • 用自动化测试和日志/指标证明结果，不只看“接口返回成功”。

学习记录保留四类内容：概念图、最小可运行示例、失败案例、项目内对应代码位置。

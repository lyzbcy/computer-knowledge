二、下载测试用例生成的提示词工程

在 codex 客户端上加载工程。

创建项目

跳转到一个空目录，执行以下命令：

cd /Users/zhangsan/workspace/codex
git clone https://gitlab.oa.com/wshoto-qa/claudecodeprompts.git

为了保护我们的仓库安全，建议设置 codex workspace 在 claudecodeprompts 工程目录下。

三、编写测试用例

1. 创建一个项目，在 codex 里编写测试用例

在 codex 中创建一个工程用于编写测试用例。

2. 输入 prompt

项目背景：

我们是微盛，需要给小程序和 APP 进行自动化测试用例生成与执行。本工程用于测试用例生成与执行的实践验证。

项目结构：

  • /codex （工程根目录）
  • /codex/prompt （prompt 根目录）
  • /codex/prompt/test-case （测试用例 prompt 目录）
  • /codex/prompt/test-case/demo-h5 （测试用例 demo 目录）
  • /codex/prompt/test-case/demo-h5/test-cases （测试用例目录）

接下来，需要给它一个 PRD 作为输入。这个 PRD 我们先放到 /codex/prompt/test-case/demo-h5/ 目录下。当前目录下已经有了 huawei_phase5_prd.md 文件，它就是一个 PRD 示例。

3. 输入 PRD 文件路径

在 codex 中输入以下命令：

请基于 PRD 生成测试用例，PRD 路径：/codex/prompt/test-case/demo-h5/huawei_phase5_prd.md

四、生成测试用例

1. 生成测试用例

已根据 PRD 生成测试用例产物：

  • 功能点大纲：功能点大纲.md
  • 测试用例：测试用例.xlsx

本次共拆解 41 个功能点，生成 58 条测试用例。XLSX 已校验包含两个 sheet：测试用例、追踪矩阵，并按你的规范保留了「测试结果」和「测试执行过程」为空，便于后续执行后回填。

📄 功能点大纲.md（文档 · MD） 📊 测试用例.xlsx（电子表格 · XLSX）

2. 工作流程

  1. 读取 PRD 全文和图片内容。
  2. 按页面、模块、业务流程拆分需求。
  3. 生成测试大纲，包含功能点清单、测试场景矩阵和风险点。
  4. 基于测试大纲和执行结果（如有），迭代更新测试用例。
  5. 生成最终测试用例文件，格式为 xlsx，并包含追踪矩阵 sheet。

生成测试用例的截图示例如下，实际功能点远比截图多。

工程根目录的 test-cases 文件夹：

打开可以看见，生成的测试用例都在 test-cases 文件夹下，以「项目名-模块名-年月日」格式命名。

例如：

demo-h5-首页-20250101.xlsx

实际命名会有细微差异，请以实际生成的文件名准。

这是 PRD 的处理示例：这是将 PRD 转化为功能点拆解和测试用例的示例。

五、执行测试用例

1. 找到执行命令文件

在 codex 中找到对应执行命令的文件。

2. 部署 odset service 和项目代码

请使用 Claude code 部署 odset service 和项目代码：

  • 项目代码：https://gitlab.oa.com/wshoto-qa/odsetestservice
  • 接口文档：https://gitlab.oa.com/wshoto-qa/odsetestservice/-/blob/master/README.md
注：服务名称截图辨识为「odset service」（对应仓库 odsetestservice），若与实际拼写不符请以仓库为准。

3. 复制测试用例并执行

odset service 部署成功后，把上面 test-cases 文件夹下生成的测试用例复制到 odset service 对应的 test-cases 目录下，然后点击执行即可。
